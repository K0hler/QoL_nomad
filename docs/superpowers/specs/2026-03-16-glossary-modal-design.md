# Справочник показателей — спецификация

## Цель

Дать пользователю возможность понять, что означает каждая метрика (индекс стоимости жизни, индекс аренды и т.д.) через отдельное модальное окно-справочник.

## Решение

Модальное окно со всеми метриками, сгруппированными по секциям. Каждая метрика — название + краткое определение (1-2 предложения: что это, как читать шкалу). Открывается кнопкой в фильтр-баре.

## Компоненты

### 1. Расширение `MetricDefinition` (types/country.ts)

Добавить поле `description: string` в интерфейс `MetricDefinition`. Заполнить описания для каждой метрики в массиве `METRICS`.

Примеры описаний:
- **Индекс стоимости жизни**: «Относительная стоимость потребительских товаров и услуг. Нью-Йорк = 100. Ниже — дешевле.»
- **Индекс аренды**: «Сравнительная стоимость аренды жилья. Нью-Йорк = 100. Ниже — дешевле.»
- **Индекс качества жизни**: «Комплексная оценка условий жизни: безопасность, здравоохранение, климат, экономика. Выше — лучше.»

### 2. Кнопка в FilterBar (components/FilterBar.tsx)

- Расположение: после ползунков (внизу фильтр-бара)
- Текст: «ⓘ Справочник показателей»
- Принимает колбэк `onGlossaryOpen` через пропсы

### 3. GlossaryModal (новый компонент: components/GlossaryModal.tsx)

- Аналогичен `CountryModal` по структуре и стилям (модальное окно, overlay, закрытие по Esc и клику вне)
- Переиспользует `METRICS`, `SECTION_ORDER`, `SECTION_LABELS` из `types/country.ts`
- Группирует метрики по секциям, для каждой выводит `label` и `description`
- Не принимает данных о странах — только справочная информация

### 4. Состояние в App.tsx

- Новый стейт `showGlossary: boolean`
- Передаёт `onGlossaryOpen` в `FilterBar`
- Рендерит `GlossaryModal` при `showGlossary === true`

## Стилизация

- Переиспользовать существующие CSS-классы: `.modal-overlay`, `.modal-content`, `.modal-close`, `.modal-title`, `.modal-section`, `.modal-grid`
- Для описаний метрик: использовать существующий `.stat-label` для названия, новый класс `.metric-description` для текста определения (цвет `var(--text-muted)`, мелкий шрифт)

## Что НЕ входит в скоуп

- Tooltips при наведении на метрики в карточках/таблицах
- Поиск/фильтрация внутри справочника
- Ссылки на источники данных и методологию расчёта
- Подробные описания (методология, примеры значений)
