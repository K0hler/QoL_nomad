# Справочник показателей — спецификация

## Цель

Дать пользователю возможность понять, что означает каждая метрика (индекс стоимости жизни, индекс аренды и т.д.) через отдельное модальное окно-справочник.

## Решение

Модальное окно со всеми метриками, сгруппированными по секциям. Каждая метрика — название + краткое определение (1-2 предложения: что это, как читать шкалу). Открывается кнопкой в фильтр-баре.

## Компоненты

### 1. Расширение `MetricDefinition` (types/country.ts)

Добавить обязательное поле `description: string` в интерфейс `MetricDefinition`. Заполнить описания для каждой метрики в массиве `METRICS`.

Описания всех метрик:

**Стоимость жизни:**
- **Индекс стоимости жизни** — Относительная стоимость потребительских товаров и услуг. Нью-Йорк = 100. Ниже — дешевле.
- **Индекс аренды** — Сравнительная стоимость аренды жилья. Нью-Йорк = 100. Ниже — дешевле.
- **Индекс продуктов** — Сравнительная стоимость продуктов питания. Нью-Йорк = 100. Ниже — дешевле.
- **Индекс ресторанов** — Сравнительная стоимость питания в ресторанах. Нью-Йорк = 100. Ниже — дешевле.
- **Покупательная способность** — Соотношение средних зарплат к ценам на товары. Нью-Йорк = 100. Выше — больше можно купить.
- **Средняя аренда (мес.)** — Средняя стоимость аренды жилья в долларах за месяц.
- **Средняя зарплата (мес.)** — Средняя месячная зарплата в долларах.

**Качество жизни:**
- **Индекс качества жизни** — Комплексная оценка условий жизни: безопасность, здравоохранение, климат, экономика. Выше — лучше.
- **Безопасность** — Уровень личной безопасности и криминальной обстановки. Выше — безопаснее.
- **Здравоохранение** — Качество и доступность медицинских услуг. Выше — лучше.
- **Покупательная способность (QoL)** — Покупательная способность из индекса качества жизни. Выше — больше можно купить на среднюю зарплату.
- **Цена недвижимости / доход** — Соотношение стоимости недвижимости к годовому доходу. Ниже — доступнее жильё.
- **Время в пробках (мин)** — Среднее время в пути на работу в одну сторону. Ниже — меньше времени в дороге.
- **Загрязнение** — Уровень загрязнения окружающей среды. Ниже — чище воздух и среда.

**Визы и пребывание:**
- **Нужна виза** — Требуется ли виза для въезда (для граждан РФ).
- **Макс. срок пребывания (дни)** — Максимальное количество дней безвизового пребывания или по визе.
- **Digital Nomad виза** — Наличие специальной визы для удалённых работников.

**Инфраструктура:**
- **Скорость интернета (Mbps)** — Средняя скорость фиксированного интернета в мегабитах в секунду.
- **Стоимость интернета ($/мес)** — Средняя ежемесячная стоимость интернета в долларах.
- **Население (млн)** — Численность населения страны в миллионах.
- **Официальный язык** — Основной официальный язык страны.
- **Уровень английского** — Уровень владения английским среди населения (high — высокий, medium — средний, low — низкий). Выше — проще общаться на английском.

**Климат:**
- **Индекс климата** — Комплексная оценка климата: температура, влажность, количество осадков и солнечных дней. Выше — комфортнее.
- **Средняя температура (°C)** — Среднегодовая температура воздуха в градусах Цельсия.
- **Солнечных дней в году** — Среднее количество солнечных дней за год.

**Налоги:**
- **Ставка подоходного налога** — Базовая ставка подоходного налога для физических лиц.

### 2. Кнопка в FilterBar (components/FilterBar.tsx)

- Расположение: после ползунков (внизу фильтр-бара)
- Текст: «ⓘ Справочник показателей»

### 3. GlossaryModal (новый компонент: components/GlossaryModal.tsx)

- Аналогичен `CountryModal` по структуре и стилям (модальное окно, overlay, закрытие по Esc и клику вне)
- Переиспользует `METRICS`, `SECTION_ORDER`, `SECTION_LABELS` из `types/country.ts`
- Группирует метрики по секциям, для каждой выводит `label` и `description`
- Не принимает данных о странах — только справочная информация

### 4. Состояние в App.tsx

- Новый стейт `showGlossary: boolean`
- Передаёт `onGlossaryOpen` в `FilterBar`
- Рендерит `GlossaryModal` при `showGlossary === true`

## Структура разметки GlossaryModal

Внутри модалки — вертикальный список, НЕ grid. Каждая секция:

```
.modal-section.section-{sectionKey}
  h3 (заголовок секции из SECTION_LABELS)
  .glossary-list (вертикальный список)
    .glossary-item (для каждой метрики)
      .glossary-term (название метрики — label)
      .glossary-def  (описание метрики — description)
```

Не использовать `.modal-grid` и `.mini-card` — они предназначены для пар label+value в карточках стран.

## Стилизация

- Переиспользовать: `.modal-overlay`, `.modal-content`, `.modal-close`, `.modal-title`, `.modal-section`
- Новые классы:
  - `.glossary-list` — вертикальный список без grid
  - `.glossary-item` — отступ между метриками
  - `.glossary-term` — название метрики, `font-weight: 600`
  - `.glossary-def` — описание, `color: var(--text-muted)`, `font-size: 0.85rem`

## Поведение модалки

- Закрытие по `Esc` и клику на overlay (аналогично `CountryModal`)
- Блокировка скролла body (`overflow: hidden`) при открытии, восстановление при закрытии
- `aria-label="Справочник показателей"` на dialog
- Модалки управляются независимым стейтом. При открытии справочника другие модалки не закрываются принудительно — overlay справочника перекроет их. На практике пользователь не может открыть справочник, пока открыта карточка страны (кнопка в фильтр-баре видна всегда, но перекрыта overlay карточки)

## Что НЕ входит в скоуп

- Tooltips при наведении на метрики в карточках/таблицах
- Поиск/фильтрация внутри справочника
- Ссылки на источники данных и методологию расчёта
- Подробные описания (методология, примеры значений)
