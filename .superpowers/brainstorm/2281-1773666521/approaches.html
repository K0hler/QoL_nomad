Архитектурный подход

Как организовать код для модалки, сравнения и новых данных

A

Компонентный подход (рекомендую)

Разбить App.tsx на отдельные компоненты:

  • CountryCard — карточка страны + чекбокс для сравнения
  • CountryModal — модалка с полной информацией
  • ComparisonPanel — плавающая панель "Сравнить N стран"
  • ComparisonTable — таблица сравнения бок о бок
  • FilterBar — существующие фильтры

✓ Плюсы

  • Чистая структура, легко поддерживать
  • Каждый компонент тестируется отдельно
  • TypeScript strict mode проще соблюдать

✗ Минусы

  • Нужно продумать передачу стейта между компонентами
  • Больше файлов в проекте
B

Минимальный подход — всё в App.tsx

Добавить модалку и сравнение прямо в существующий App.tsx:

  • Новые useState для selectedCountry, comparedCountries, showComparison
  • JSX для модалки и таблицы сравнения inline
  • Весь CSS в index.css

✓ Плюсы

  • Быстрее реализовать
  • Нет проблем с пробросом пропсов
  • Меньше файлов

✗ Минусы

  • App.tsx вырастет до 400-500 строк
  • Сложнее читать и поддерживать
  • Смешение ответственности в одном файле
